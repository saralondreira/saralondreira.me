## Overview

A centralized web-based Educational Management Platform being developed as my final project for the IEFP Software Engineering specialization. The system is designed to digitalize, optimize, and secure the administrative and pedagogical workflows of professional training institutions, specifically aligning with the modernization guidelines of European-funded programs (like PESSOAS 2030).

[**View Code on GitHub ↗**](https://github.com/saralondreira/Plataforma-Gestao-Pedagogica.git)

![Platform Login Interface](/images/projects/projects/pedagogical-platform/login.png)

## The Challenge

Before this platform, the administrative processes suffered from critical inefficiencies:
- **Hardware Dependency & Logistics:** Mandatory physical delivery of pedagogical documentation via CDs/DVDs, causing logistical bottlenecks and compatibility issues with modern hardware.
- **Information Fragmentation:** Disorganized use of disparate digital tools (Drive, Teams, Email) created information silos, making it impossible to automatically track document submissions and compliance.
- **Data Security & GDPR Risks:** High risk of data loss due to physical damage of media or lack of centralized backups, alongside severe GDPR compliance risks regarding trainee data circulation.

## The Solution

I architected a **Unified Pedagogical Platform** featuring hierarchical role-based access (Trainees, Trainers, and Coordinators/Mediators). The system centralizes communication, pedagogical documentation, and daily activity tracking into a single secure environment.

![Platform Control Panel](/images/projects/projects/pedagogical-platform/controlpanel.png)

### Key Features & Business Logic

- **Smart Scheduling & Calendar ("The Killer Feature"):** An interactive intelligent calendar allowing trainers to manage their availability (Free/Busy slots), streamlining the complex process of scheduling classes across different modules and courses.
- **Approval Workflows & State Management:** Implemented strict validation states (e.g., "Submitted", "Validated by Mediator", "Rejected") for daily summaries and document deliveries, ensuring data integrity before final institutional approval.

![Coordinator Dashboard](/images/projects/projects/pedagogical-platform/dashboard.png)

- **Compliance & Tracking Dashboard:** An automated dashboard for coordinators that tracks delivery statuses. The algorithm actively checks deadlines against module completion dates and flags missing documentation in real-time, drastically reducing non-compliance risks during European funding audits.
- **Attendance & Daily Activity Logging:** Dedicated modules for trainers to submit daily class summaries, log hours taught, and register student attendance.

![Class Management View](/images/projects/projects/pedagogical-platform/turma.png)

## Technical Architecture & Stack

The project is built with a full-stack approach, following a structured software development lifecycle:
- **Frontend:** HTML5, CSS3 (Grid/Flexbox), and JavaScript for a responsive user interface.
- **Backend:** Node.js, handling secure file reception, renaming logic, and server-side operations.
- **Database:** Relational Database (SQL) mapped with complex entity relationships.
