## Overview

A console-based Java application designed to simulate and manage the complex daily operations of a hospital's emergency department. Developed as a final group project for the Project Laboratory 1 (LAPR1) course at ISEP, this system emphasizes deep understanding of Object-Oriented Programming (OOP) principles, algorithmic logic for triage, and low-level data structure management without relying on dynamic collections.

[**View Code on GitHub ↗**](https://github.com/saralondreira/Javolt-Hospital-ISEP.git)

![Hospital System Welcome Screen](/images/projects/projects/hospital-management/welcome.png)

## Key Features

- **Dynamic Time Simulation:** Custom implementation of time units (24 per day) to progress hospital operations, managing doctor shifts, tracking patient waiting times, and automatically escalating urgency levels based on wait duration.
- **Smart Triage & Automated Routing:** An intelligent algorithm that calculates a patient's urgency level and automatically assigns them to the correct medical specialty based on their dynamically inputted symptoms.

![Hospital System Main Menu](/images/projects/projects/hospital-management/menu.png)

- **Entity Management & Statistics:** Full management of doctors, patients, and specialties, alongside the generation of critical daily statistics (e.g., average patients per day, doctor daily earnings, and top specialties).
- **Data Persistence:** Robust File I/O system to load initial state configurations and persistently save all operational history, inputs, and outputs safely to text files.

![Hospital Configuration Settings](/images/projects/projects/hospital-management/config.png)

## Technologies & Concepts Used

- **Programming Language:** Java (Console Application)
- **Core Concepts:** Object-Oriented Programming (Inheritance, Polymorphism, Encapsulation, Abstraction), File I/O
- **Data Structures:** Strict use of raw Arrays (dynamic lists like ArrayLists were strictly forbidden to challenge memory management skills).
- **Version Control & Collaboration:** Git & GitHub (Agile collaboration using feature branches and Pull Requests).
